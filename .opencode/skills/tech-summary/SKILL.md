# 技术分析与摘要技能

## 使用场景

当需要对采集的技术内容进行深度分析总结时使用此技能。基于原始采集数据，逐条完成摘要提炼、技术亮点挖掘、评分与标签建议，并输出结构化分析结果。

## 执行步骤

1. **读取采集文件**：读取 `knowledge/raw/` 目录下当日最新的采集 JSON 文件
2. **逐条深度分析**：
   - 为每条撰写中文摘要（50-200字）
   - 提取 2-5 个技术亮点，用事实说话，不编造
   - 评分（1-10），按 tech_depth / innovation / usability 三维打分，附评分理由
   - 给出标签建议（3-5 个）
3. **趋势发现**：基于全部项目，识别共同主题与新兴概念，写入趋势分析字段
4. **输出分析结果 JSON**：将分析结果以标准 JSON 格式输出，待 Organizer 写入

## 评分标准

| 分数区间 | 等级 | 说明 |
|----------|------|------|
| 9-10 | 改变格局 | 突破性技术、范式转变、引领行业趋势 |
| 7-8  | 直接有帮助 | 实用性强、直接提升工作效率或生活质量 |
| 5-6  | 值得了解 | 有趣但非必需，了解有益 |
| 1-4  | 可略过 | 重复性高、适用面窄或价值不明显 |

## 注意事项

- 评分必须有依据，基于内容原文分析得出
- 摘要使用中文，50-200字
- 技术亮点只陈述事实，不做过度推断
- 标签需覆盖技术领域、应用场景、重要程度
- **评分约束**：单次分析的 20 个项目中，9-10 分项目不超过 2 个
