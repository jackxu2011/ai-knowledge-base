# AI 知识库 · 项目愿景 v1.0

## 要做什么

- 每天抓取 GitHub Trending（最多 20 条，先关键字过滤 AI 相关，再交大模型评审，不补充非 Trending 内容）
- 用 Agent 分析内容（自动爬取项目 README 或 Hacker News 原文，输出结构化字段：摘要、关键点、技术深度、技术类别、创新点、使用难度打分及加权最终得分）
- 输出知识条目（JSON 格式，沿用 AGENTS.md 定义的 12 字段标准结构，新增多维度评分字段：技术类别及打分、创新点打分、使用难度打分；加权平均值最终得分以 `score` 字段平铺到最外层）

## 不做什么

- 不做通用爬虫（产品定位选择，非技术限制；架构预留扩展接口，未来可接入 ArXiv/Reddit 等数据源而无需推翻重来）
- 不做内容审核（依赖数据源自身质量：GitHub Trending 和 Hacker News 不会出现违规内容或低端错误；内容分析环节由下游 Agent 处理）
- 不存数据库（使用文件系统存储 JSON）

## 边界 & 验收

### 失败处理

- 单次失败：重试 3 次后放弃，跳过该条数据（不回滚其他成功数据）
- 多次失败：连续失败发邮件告警通知
- 超时机制：整个 pipeline 超过 10 分钟强制终止

### 性能要求

- 每天处理时间控制在 10 分钟内
- 定时任务：凌晨 3 点（服务器空闲）运行，不影响次日分发时效性

### 成本控制

- 大模型 token 成本：先运行观察后再定预算
- 超预算策略：继续运行并发邮件告警
- 监控方式：每次运行完后统计 token 消耗

### 质量保证

- 工具自动化验证项目可运行（能启动不报错）
- JSON Schema 校验必须字段完整性
- 模型多维度评分 + 前期人工 review，持续迭代优化评分标准

## 怎么验证

- **可运行性**：自动化测试验证项目能启动不报错；若生成的 JSON 为空或格式错误，视为不可运行
- **数据完整性**：JSON Schema 强制校验 12 个标准字段 + 多维度评分字段，确保必要内容不缺失（summary 由 AI 生成，理论不为空）；加权平均值最终得分以 `score` 字段平铺到最外层
- **内容质量**：模型打分过滤（技术类别、创新点、使用难度）+ 前期人工 review 双重保障；通过持续迭代模型评分标准优化长期质量
